Privacy Policy

Effective date: 2025-11-07

Ren is a community Discord bot that provides a fictional in-server economy, social interactions, server utilities, and image-processing features. This privacy policy explains what information Ren may collect, how it is used, how long it is retained, and the choices available to server owners and users.

Data controller & contact

Maintainer: Ethan H. Lawr
Contact: ethan.h.lawr@gmail.com

What information we collect

We collect only the minimum information necessary to provide bot features:

  • Discord identifiers: user IDs and server (guild) IDs to link stored state and configuration.
  • User-supplied content: text or media provided directly to the bot (command arguments, images/attachments for processing).
  • Command metadata: command name, timestamp, user ID, and other context needed to execute commands.
  • Diagnostic logs: non-sensitive runtime logs used for debugging; these may include short snippets of message content only when necessary to diagnose an error.

How we use the data

  • Execute and respond to commands (economy, image processing, utilities).
  • Persist user/server state (balances, inventory, timers, configuration).
  • Process images and deliver generated results; temporary files are discarded after delivery unless a feature explicitly saves them.
  • Aggregate and analyze errors and usage for reliability and improvements.
  • Comply with legal requests where required by law.

Message content: when and why it is read

Ren reads message content only when explicitly invoked by a user:

  • Messages that begin with the configured prefix and are directed at the bot.
  • Messages that mention the bot directly.
  • Direct messages (DMs) sent to the bot.

Message content is used only to parse commands, extract free-text arguments (amounts, URLs), and locate inline image URLs or attachments for image-processing commands. Ren does not perform passive or bulk scanning of server messages.

Storage & retention

  • Persistent state (user balances, inventory, preferences): retained until deletion or an authorized data removal request.
  • Diagnostic logs: kept for a short, reasonable period (28 days) to assist debugging.
  • Temporary files (generated images): deleted after processing and delivery unless explicitly saved.

Self-hosted deployments: administrators that self-host the bot control data retention and storage location.

Disclosure & sharing

Ren does not sell or rent personal information. Third-party services (e.g., hosted databases, error tracking) may process data on our behalf; we require reasonable security practices from service providers. Ren will disclose data to comply with lawful requests.

Security

We use commercially reasonable measures to protect stored data. In the event of a security incident we will take steps to mitigate and notify affected parties where required by law.

User controls & choices

  • Server admins can disable prefix commands and require slash commands to avoid message-content usage.
  • Users may prefer to DM the bot when sending sensitive input.
  • Data access, correction, or deletion requests: contact ethan.h.lawr@gmail.com with server/user IDs and requested actions.

Children

Ren is not intended for users under Discord's minimum age. We do not knowingly collect personal information from children. If you believe a child has provided personal data, contact the maintainer and we will take steps to delete it.

Changes to this policy

We may update this policy to reflect changes. We will publish the effective date and summarize material changes.
